policy : 보안을 위해 달성해야 하는 목적
mechanism : policy 구현을 위해 사용하는 것들
assurance : 각 항목에 대한 신뢰 정도
incentive : 사용자의 동기

secrecy : 특정한 정보에 접근할 수 있는 개체의 수를 제한하는 것
Privacy : 개인적인 정보를 통제하는 권리 및 능력
confidentiality(기밀성) : 누군가의 비밀을 보호해야할 의무
Authenticity : 확실성, 진실성의 개념
Integrity : 무결성, 조작하지 않았다는 보장

hack : 시스템의 룰을 위반하지 않지만, 설계자가 원하지 않은, 의도하지 않은 행동

Anonymity(익명성) : 메타데이터에 접근을 제한하는 것, 주체를 식별할 수 없는 것 , 행동과 주체를 연결할 수 없는 것


Security policy : 보호하고자 하는 Protection 전략에 대한 설명, 명시
Security target : 특정 시스템 및 물체에 대해 Protection 전략을 구현하기 위한 수단을 명시
protection profile : 서로 다른 시스템에 대한 비교가 가능하도록 일반적으로 작성한 것

vulnerability**(취약점) : 내부 및 외부의 위협, 시스템 자체의 보안 정책을 위반할 수 있는 점 -> 공략 성공하면 보안 실패로 이어짐

보안의 3대 요소 CIA - 
Confidentiality(기밀성) : 허가되지 않은 사람이 특정 정보에 접근하는 것을 방지하는 것
Integrity(무결성) : 허가되지 않은 사람이 정보를 변경하거나 작성하는 것을 방지하는 것
Availability(가용성) : 자료 및 정보를 필요할 때 사용 가능해야 함
